# Excel 导入模板说明

## 模板文件列表

本项目提供三个标准 Excel 导入模板，用于导入不同类型的数据：

1. **订单数据模板**: `templates/orders_template.xlsx`
2. **核销记录模板**: `templates/verification_records_template.xlsx`
3. **收银流水模板**: `templates/cash_flow_records_template.xlsx`

---

## 模板使用规范

### 通用规则

1. **文件格式**: 支持 `.xlsx` 和 `.csv` 格式
2. **编码格式**: UTF-8（CSV 文件必须使用 UTF-8 编码）
3. **第一行**: 必须是表头（字段名），从第二行开始是数据
4. **必填字段**: 标记为"必填"的字段不能为空
5. **日期格式**: 统一使用 `YYYY-MM-DD HH:mm:ss` 格式（如：2026-07-11 14:30:00）
6. **金额格式**: 数字类型，保留两位小数（如：99.90）
7. **文本字段**: 避免包含特殊字符（如换行符、制表符）

## 模板 1：订单数据模板 (orders_template.xlsx)

### 表头字段说明

| 列号 | 字段名 | 中文说明 | 是否必填 | 数据类型 | 示例值 | 备注 |
|------|--------|----------|----------|----------|--------|------|
| A | order_id | 订单号 | ✅ 必填 | 文本 | DY20260711001 | 唯一标识，不能重复 |
| B | store_id | 门店ID | ✅ 必填 | 文本 | STORE_001 | 门店唯一标识 |
| C | store_name | 门店名称 | ✅ 必填 | 文本 | 北京朝阳店 | - |
| D | product_name | 商品名称 | ✅ 必填 | 文本 | 双人套餐A | - |
| E | amount | 订单金额 | ✅ 必填 | 数字 | 199.00 | 单位：元，保留2位小数 |
| F | order_time | 下单时间 | ✅ 必填 | 日期时间 | 2026-07-11 12:30:00 | YYYY-MM-DD HH:mm:ss |
| G | status | 订单状态 | ✅ 必填 | 文本 | completed | completed/cancelled/refunded |
| H | source | 数据来源 | ⭕ 可选 | 文本 | douyin | douyin/meituan/manual，默认manual |

### 示例数据

```
order_id,store_id,store_name,product_name,amount,order_time,status,source
DY20260711001,STORE_001,北京朝阳店,双人套餐A,199.00,2026-07-11 12:30:00,completed,douyin
DY20260711002,STORE_001,北京朝阳店,单人午餐,59.90,2026-07-11 13:15:00,completed,douyin
DY20260711003,STORE_002,上海浦东店,家庭套餐,299.00,2026-07-11 14:00:00,completed,meituan
```

## 模板 2：核销记录模板 (verification_records_template.xlsx)

### 表头字段说明

| 列号 | 字段名 | 中文说明 | 是否必填 | 数据类型 | 示例值 | 备注 |
|------|--------|----------|----------|----------|--------|------|
| A | order_id | 订单号 | ✅ 必填 | 文本 | DY20260711001 | 必须与订单数据中的order_id对应 |
| B | store_id | 门店ID | ✅ 必填 | 文本 | STORE_001 | 必须与订单数据中的store_id一致 |
| C | verify_time | 核销时间 | ✅ 必填 | 日期时间 | 2026-07-11 18:30:00 | YYYY-MM-DD HH:mm:ss |
| D | verify_amount | 核销金额 | ✅ 必填 | 数字 | 199.00 | 单位：元，保留2位小数 |
| E | operator | 操作员 | ⭕ 可选 | 文本 | 张三 | 核销人员姓名或工号 |
| F | source | 数据来源 | ⭕ 可选 | 文本 | douyin | douyin/meituan/manual，默认manual |

### 示例数据

```
order_id,store_id,verify_time,verify_amount,operator,source
DY20260711001,STORE_001,2026-07-11 18:30:00,199.00,张三,douyin
DY20260711002,STORE_001,2026-07-11 19:00:00,59.90,李四,douyin
DY20260711003,STORE_002,2026-07-11 20:15:00,299.00,王五,meituan
```

### 注意事项

⚠️ **重要**: 
- 一个订单可能被多次核销（重复核销场景），系统会自动检测
- 核销金额应与订单金额一致，不一致会被标记为"金额不匹配"差异

## 模板 3：收银流水模板 (cash_flow_records_template.xlsx)

### 表头字段说明

| 列号 | 字段名 | 中文说明 | 是否必填 | 数据类型 | 示例值 | 备注 |
|------|--------|----------|----------|----------|--------|------|
| A | order_id | 订单号 | ⭕ 可选 | 文本 | DY20260711001 | 如有则填，部分POS流水无订单号 |
| B | store_id | 门店ID | ✅ 必填 | 文本 | STORE_001 | 门店唯一标识 |
| C | amount | 收款金额 | ✅ 必填 | 数字 | 199.00 | 单位：元，保留2位小数 |
| D | record_time | 流水时间 | ✅ 必填 | 日期时间 | 2026-07-11 18:35:00 | YYYY-MM-DD HH:mm:ss |
| E | payment_method | 支付方式 | ✅ 必填 | 文本 | wechat | wechat/alipay/cash/card |
| F | remark | 备注 | ⭕ 可选 | 文本 | 团购核销 | 额外说明信息 |
| G | source | 数据来源 | ⭕ 可选 | 文本 | pos_system | 收银系统名称，默认manual |

### 示例数据

```
order_id,store_id,amount,record_time,payment_method,remark,source
DY20260711001,STORE_001,199.00,2026-07-11 18:35:00,wechat,团购核销,pos_system
DY20260711002,STORE_001,59.90,2026-07-11 19:05:00,alipay,团购核销,pos_system
,STORE_002,150.00,2026-07-11 20:30:00,cash,散客消费,pos_system
```

### 注意事项

⚠️ **重要**:
- 部分收银系统导出的流水可能没有订单号，这是正常现象
- 系统会通过"时间窗口+金额+门店"进行模糊匹配
- 同一笔订单可能有多条流水记录（分次付款），系统会检测异常

## 常见错误及解决方案

### 错误 1：日期格式不正确

**错误示例**: `2026/7/11 12:30` 或 `07-11-2026`

**正确格式**: `2026-07-11 12:30:00`

**解决方案**: 在 Excel 中选中日期列 → 右键"设置单元格格式" → 选择"自定义" → 输入 `yyyy-mm-dd hh:mm:ss`

---

### 错误 2：金额包含货币符号

**错误示例**: `¥199.00` 或 `$199.00`

**正确格式**: `199.00`（纯数字）

**解决方案**: 使用查找替换功能删除货币符号，或设置单元格格式为"数值"

---

### 错误 3：订单号重复

**错误提示**: "订单号 DY20260711001 已存在，跳过导入"

**解决方案**: 检查 Excel 中是否有重复的订单号，或在导入前清理历史数据

---

### 错误 4：必填字段为空

**错误提示**: "第 5 行缺少必填字段：order_id"

**解决方案**: 补充缺失的必填字段后重新导入

---

### 错误 5：文件编码问题（CSV）

**错误提示**: "解析失败，检测到非UTF-8编码"

**解决方案**: 
1. 用记事本打开 CSV 文件
2. 点击"文件" → "另存为"
3. 编码选择 "UTF-8"
4. 保存后重新上传

---

## 最佳实践建议

1. **导入前备份**: 每次导入前导出当前数据作为备份
2. **小批量测试**: 首次使用时先导入 10-20 条数据测试
3. **批次管理**: 每次导入使用不同的批次号，便于追溯
4. **数据校验**: 导入后查看导入报告，确认成功/失败记录数
5. **定期清理**: 删除过期的测试数据，保持数据库整洁
